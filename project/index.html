Swarm Computing

The particle swarm optimization was originally presented by Kennedy and Eberhart, it has been compared with Genetic Algorithm, it is inspired by social behavior among individuals, these individuals are moving through an n-dimensional search space, each particle represents a potential solution of the problem, and can remember the best position it has reached. All the particles can share their information about the search space, so there is a global best solution. In each of iteration, every particle calculates its velocity according to the formula. The basic PSO model consists of a swarm of particles, which are initialized with a population of random candidate solutions. They move iteratively through the d-dimension problem space to search for the new solutions, where the fitness, f, can be calculated as the certain qualities measure. Each particle has a position represented by a position-vector xi (i is the index of the particle), and a velocity represented by a velocity-vector vi. Each particle remembers its own best position so far in a vector i-th, and its d- dimensional value is pbest(pid). The best position-vector among the swarm so far is then stored in the vector i-th, and its d-th dimensional value is gbest(pgd).

Day Care

It had 3 active users namely Parents/Guardians, Teachers/Helpers and Admin. Financial Aid

The application was developed to aid underprivileged students with their financial needs to continue their education by allowing them to connect with available sponsors who wish to provide open or closed sponsorship to the students as per the availability. It has 5 major players Student, Sponsor, Manager, Bank and University and each player has defined set of roles. Students looking for sponsorships should create a profile, providing their information including proof of financial need, proof of acceptance from University, which will be created upon approval from the Admin once the documents have been verified. There are two types of scholarships, open and closed. The open scholarships would allow the student to make a fund-raising request, and the closed scholarship would allow the student to apply for specific available/multiple sponsorships from the University. The students is allowed to select either of these while creating their profiles. The student can also apply for a loan from the bank if his financial needs are not covered with the sponsorships. Sponsors who wish to help these students can create a profile and help the students accordingly. The Admin controls everything and approves the documents as well as profiles of bank, student and universities. He also has access to a dashboard which provides details about students and sponsors.

Yellow Pages

The purpose of this application is to create a website using MVC architecture and hibernate database to facilitate trading of second hand books between peers. The website will allow users to sign-in and create an account. Once the account is created the user can browse through all the books available and either choose to trade for another book or buy the book from the owner. A single book could be traded multiple times and there are no limitations for the category of books. User will be able to select from multiple category and trade books within a category.The books are added by admin and are modifiable by admin themselves. The admin portal can only be accessed by authorized users which adds a layer of security to the application. All thebooks are stored in mySQL database using hibernate and are persisted using hibernate annotations. Some of the technologies used include Spring Security to authorize admin login, Angular js to add data to cart and view cartdata, Hibernate Annotations to persist book objects to table and retrieve them, CRUD operations performed by the admin to modify or delete the persisted data, Use of JSTL tags to include code logic into jsp pages and Use of annotation mapping to redirect users to appropriate page to name a few.
